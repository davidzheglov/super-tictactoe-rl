# Research Training Plan

This plan is for the second-stage experiment after the first transferred
checkpoints failed to beat the smart heuristic.

## Baseline Result To Explain

The first completed run is recorded in `REPORT_RESULTS.md`:

- PPO/TorchRL, 6,000 stochastic episodes: 6% win rate against the smart heuristic.
- DQN, 6,000 stochastic episodes: 0% win rate against the smart heuristic.

This is a useful negative baseline. It shows that sparse-reward neural training
from scratch is not enough against a strong tactical policy.

## Decisions For The Long Run

The next run uses three anti-sparse-reward techniques.

1. Behavior cloning before RL.
   The policy first imitates a mixed tactical teacher dominated by the smart
   heuristic. This gives PPO a human-knowledge starting point instead of a blank
   random policy.

2. Mid-game state starts.
   PPO episodes can begin from scripted non-empty positions generated by the
   smart heuristic, line-builder, and random play. This forces the network to
   learn tactical decisions from many board states, not only from opening moves.

3. Checkpoint selection.
   Every save interval also writes a numbered checkpoint. After training, all
   saved checkpoints should be benchmarked against the smart heuristic and line
   builder. The best checkpoint, not necessarily the final checkpoint, should be
   used in the report and Pygame demo.

## Episode Counts

The remote server completed the previous neural run quickly enough that the
second-stage PPO experiment is set much larger:

- behavior cloning: 200,000 teacher positions, 8 epochs;
- stochastic PPO/TorchRL: 48,000 episodes;
- deterministic-placement PPO/TorchRL: 48,000 episodes;
- optional DQN baseline: 18,000 episodes;
- optional Q-learning baseline: 20,000 episodes.

An episode means one complete game, or the remaining part of one complete game
when training starts from a non-empty board state.

The main two PPO jobs are designed to run in parallel on the two A2 GPUs. The
deterministic-placement PPO uses the same board game but removes the 1/2 direct
and 1/16 neighbour stochastic placement rule: the chosen cell is always filled
if it is legal. This is not the assignment environment, but it is useful as an
ablation and may learn a cleaner tactical prior that can later be tested in the
stochastic game.

## Main Command

Use this command when the goal is to spend the overnight budget on the most
promising agents:

```bash
python run_remote_training.py \
  --neural-backend torchrl \
  --gpus 0,1 \
  --output-dir runs/research_bc_ppo_48k \
  --only behavior_clone,ppo,ppo_deterministic \
  --enable-behavior-clone \
  --include-deterministic-ppo \
  --bc-samples 200000 \
  --bc-epochs 8 \
  --ppo-episodes 48000 \
  --deterministic-ppo-episodes 48000 \
  --ppo-batch-episodes 32 \
  --save-interval 2000 \
  --log-interval 500 \
  --ppo-opponent mixed \
  --mixed-self-prob 0.10 \
  --mixed-heuristic-prob 0.65 \
  --mixed-line-prob 0.20 \
  --mixed-basic-prob 0.00 \
  --mixed-random-prob 0.05 \
  --start-state-mode mixed \
  --start-state-min-plies 4 \
  --start-state-max-plies 18 \
  --shaping-scale 0.04 \
  --shaping-clip 2.0 \
  --shaping-defense-weight 0.85 \
  --forfeit-penalty 0.025
```

This command runs behavior cloning first, then launches:

- stochastic PPO/TorchRL on one GPU;
- deterministic-placement PPO/TorchRL on the other GPU.

## Checkpoint Benchmarking

After the PPO jobs finish:

```bash
python benchmark_checkpoints.py \
  runs/research_bc_ppo_48k/ppo_seed0/checkpoints \
  runs/research_bc_ppo_48k/ppo_deterministic_seed0/checkpoints \
  --games 200 \
  --device cpu \
  --deterministic \
  --output-csv runs/research_bc_ppo_48k/checkpoint_benchmark.csv
```

The strongest model is the checkpoint with the highest `agent_win_rate` against
the smart heuristic. Use that checkpoint for the final Pygame demo and report.
